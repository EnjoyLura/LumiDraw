## 1. 产品概述与定位

### 1.1 产品愿景

**一句话定义**：Lumi-Draw —— 人人都是AI艺术家。

**核心价值主张**：低门槛、多模型、社区驱动的AI创作平台。用户无需注册多个平台、无需了解复杂的AI技术，即可在一个小程序内体验到全球顶尖AI模型的创作乐趣，并通过画廊社区分享和发现灵感。

### 1.2 产品定位矩阵

| 维度 | 定位 |
|------|------|
| 专业度 | 面向大众（小白友好，兼顾专业用户的进阶需求） |
| 功能广度 | 综合型（创作 + 社区 + 工具三位一体） |
| 平台 | 微信小程序（轻量化，无需下载App） |
| 商业模式 | 积分预付费制 |

### 1.3 差异化优势

| 优势 | 说明 |
|------|------|
| **多模型聚合** | Google / OpenAI / ByteDance / 微信混元等主流模型一站聚合，用户无需注册多个平台 |
| **微信社交裂变** | 小程序天然适配微信生态，邀请码、作品分享、画廊社区均可利用社交关系链传播 |
| **画廊社区驱动** | "生成同款"功能降低创作门槛，社区UGC形成内容飞轮 |
| **轻量入口** | 小程序即用即走，降低用户决策成本 |

### 1.4 产品路线图

#### P0 — MVP 阶段（第1-4周）
- 创作页核心功能：Prompt 输入 + 多模型选择 + 生成预览
- 作品页：历史作品列表
- 我的页：基础用户信息 + 积分余额
- 基础积分系统：注册赠送 + 充值 + 消耗扣减
- 用户登录与注册

#### P1 — 社区与灵感阶段（第5-8周）
- 画廊页：瀑布流展示 + 分类筛选 + 生成同款 + 收藏/点赞
- 灵感页：官方模板 + AI图片处理工具
- 完整积分体系：签到 + 邀请码 + 社区贡献奖励
- 作品发布到画廊

#### P2 — 商业化与运营阶段（第9-12周）
- 会员订阅体系
- 社区互动（评论、关注）
- 高级AI工具（老照片修复、AI扩图等）
- 运营活动（主题创作挑战、排行榜）
- 广告变现接入

## 2. 竞品分析

### 2.1 2025-2026 AI图像生成市场格局

**全球市场**：Midjourney、DALL-E（OpenAI）、Stable Diffusion 三足鼎立，Google Nano Banana 系列快速崛起。

**国内市场**：百度文心一格、阿里通义万相、字节即梦AI（Seedream）、腾讯混元大模型、美图WHEE 群雄逐鹿。

**行业趋势**：
- 从单一文生图 → 多模态统一架构（生成 + 编辑 + 理解）
- 图像质量从1K → 4K 成为标配
- 微信生态：官方推出"AI应用成长计划"，提供免费混元算力（1亿Token文生文 + 1万张文生图）

### 2.2 竞品对比矩阵

| 维度 | 美图WHEE | 文心一格 | 通义万相 | 即梦AI | AI小聚 | 创客贴AI | **Lumi-Draw** |
|------|----------|----------|----------|--------|--------|----------|---------------|
| 平台 | App+Web | Web | Web+App | 小程序+Web | Web | Web | **小程序** |
| 模型来源 | 自研 | 自研(文心) | 自研(通义) | Seedream | 多模型 | 自研+三方 | **多厂商聚合** |
| 社区功能 | 强 | 弱 | 弱 | 中 | 弱 | 弱 | **强** |
| AI工具集 | 强 | 弱 | 中 | 中 | 强 | 强 | **中→强** |
| 社交裂变 | 弱 | 弱 | 弱 | 强(抖音) | 弱 | 弱 | **强(微信)** |
| 价格门槛 | 高 | 中 | 中 | 中 | 中 | 中 | **低** |
| 模型数量 | 1 | 1 | 1 | 1-2 | 3-5 | 1-2 | **6+** |

### 2.3 竞品功能拆解

**美图WHEE**：AI生图 + AI图片美化 + 成熟会员体系。优势在于图片处理能力强，用户基数大。劣势是价格高，社交属性弱。

**即梦AI（字节跳动）**：Seedream 系列模型 + 抖音生态 + 视频生成。优势在于模型质量高，抖音流量加持。劣势是依赖抖音生态，独立获客能力待验证。

**AI小聚**：多AI能力聚合（绘画/写作/音乐/视频），图生图、老照片修复等工具齐全。优势是功能全面。劣势是社区弱，用户粘性不高。

**创客贴AI**：AI设计工具集（抠图/去水印/变清晰），面向设计师。优势是工具实用。劣势是缺乏创作和社区闭环。

### 2.4 Lumi-Draw 竞争策略

1. **做聚合器而非模型商**：不自研模型，聚合各家最优模型，用户无需选择平台
2. **发挥微信社交裂变**：画廊社区 + 邀请码 + 分享传播，利用社交关系链低成本获客
3. **轻量化入口**：小程序即用即走，降低用户尝试成本
4. **混元免费额度引流**：利用微信提供的1万张免费文生图额度，降低初期运营成本，吸引用户体验

## 3. 目标用户画像

### 3.1 核心用户群

#### 画像A：社交分享型
- **年龄**：18-30岁，女性偏多
- **需求**：生成好看的AI头像/壁纸发朋友圈
- **行为**：低频高分享，对画面美感敏感，偏好一键操作
- **付费意愿**：中等，愿为"好看"付费
- **典型场景**：看到朋友分享的AI作品 → 点进小程序 → 生成同款 → 发朋友圈

#### 画像B：内容创作者
- **年龄**：20-35岁，自媒体/设计师
- **需求**：快速生成配图、封面、素材
- **行为**：高频使用，注重模型质量和可控性，会对比不同模型效果
- **付费意愿**：高，作为生产力工具愿意持续投入
- **典型场景**：需要文章配图 → 选模型 → 精细调Prompt → 批量生成 → 下载素材

#### 画像C：AI爱好者/尝鲜者
- **年龄**：18-40岁，男性偏多
- **需求**：体验不同模型效果，对比模型能力
- **行为**：会尝试所有模型，关注技术参数，喜欢分享评测
- **付费意愿**：中等，但愿为"更好的模型"付费
- **典型场景**：新模型上线 → 第一时间体验 → 对比效果 → 分享评测到画廊

### 3.2 用户旅程地图

**新用户路径**：
```
扫码/分享进入 → 浏览画廊（被吸引）→ 点击"生成同款" → 跳转创作页 → 
首次免费体验 → 积分耗尽 → 签到/邀请获取积分 → 充值转化
```

**活跃用户路径**：
```
灵感页发现模板 → 使用模板创作 → 精调参数 → 发布到画廊 → 
获得点赞/收藏 → 获得积分奖励 → 持续创作
```

### 3.3 用户需求优先级（Kano模型）

| 类型 | 需求 |
|------|------|
| **必备属性** | AI文生图、图片保存、历史记录、积分充值 |
| **期望属性** | 多模型选择、图生图、快速生成、批量生成 |
| **魅力属性** | 生成同款、画廊社区、AI图片处理工具、邀请裂变 |

| **Google** | Nano Banana Pro (Gemini 3 Pro) | 高质量，4K，高级文本渲染 | 6 |

  ├── 发布到画廊
  ├── 重新生成（相同参数）

## 5. 积分经济系统

### 5.1 积分基本规则

| 规则 | 说明 |
|------|------|
| 名称 | 积分 |
| 汇率 | **1元人民币 = 10积分** |
| 逆向兑换 | 积分不可兑换为现金 |
| 有效期 | 充值积分永久有效；赠送积分30天有效期 |

### 5.2 积分获取途径

#### 5.2.1 新用户赠送
- 首次注册：**赠送50积分**（约可生成5-10张图，足够体验所有模型）

#### 5.2.2 签到奖励

| 签到类型 | 积分奖励 |
|----------|----------|
| 每日签到 | +2积分 |
| 连续7天 | 额外+5积分 |
| 连续30天 | 额外+30积分 |

#### 5.2.3 邀请奖励
- 每位用户生成**6位专属邀请码**（字母+数字）
- 邀请人：被邀请者首次创作时，邀请人 **+20积分**
- 被邀请人：注册时额外 **+20积分**
- 防刷：同一设备/微信号仅计一次

#### 5.2.4 社区贡献奖励

| 行为 | 积分奖励 |
|------|----------|
| 作品发布到画廊 | +3积分 |
| 作品被收藏 | +1积分/次 |
| 作品被"生成同款" | +2积分/次 |

#### 5.2.5 充值购买

| 档位 | 积分 | 赠送 | 实际获得 | 折扣 |
|------|------|------|----------|------|
| ¥6 | 60 | — | 60 | — |
| ¥18 | 180 | +10 | 190 | 约9.5折 |
| ¥30 | 300 | +30 | 330 | 约9折 |
| ¥68 | 680 | +100 | 780 | 约8.7折 |
| ¥128 | 1280 | +250 | 1530 | 约8.4折 |

支付方式：微信支付。首充可设双倍活动。

### 5.3 积分消耗规则

#### 5.3.1 AI模型生成定价

| 模型 | 积分/张 | 折合人民币 |
|------|---------|-----------|
| 混元大模型（免费额度内） | 0 | 免费 |
| Nano Banana 2.0 | 3 | ¥0.3 |
| Nano Banana Pro | 6 | ¥0.6 |
| GPT-Image-1 | 8 | ¥0.8 |
| GPT-Image-2 | 15 | ¥1.5 |
| Seedream 4.0 | 4 | ¥0.4 |
| Seedream 4.5 | 6 | ¥0.6 |

#### 5.3.2 额外消耗
- 高分辨率(4K)：额外+2积分
- 批量生成：按张计费

#### 5.3.3 AI图片处理工具
- AI抠图 / 去水印 / 变清晰：**2积分/次**
- 老照片修复 / AI扩图 / 风格迁移：**3积分/次**

### 5.4 经济系统平衡

| 指标 | 目标 |
|------|------|
| 毛利率 | 60%以上（积分收入 vs AI API调用成本） |
| 免费→付费转化率 | 5-10% |
| 赠送积分占比 | 控制在总流通积分的20%以内 |
| 混元免费额度 | 每日每用户限3张，作为引流手段 |

**防控措施**：
- 赠送积分设30天有效期，防止积分囤积
- 签到/邀请产出可控，避免积分通胀
- 监控单用户异常行为（短时间大量注册/生成）

## 6. 盈利模式

### 6.1 核心收入：积分充值

- **模式**：用户充值积分 → 消耗于AI生图/工具
- **收入**：充值金额
- **成本**：AI API调用费用 + 云开发费用 + 微信支付手续费(0.6%)
- **目标毛利率**：60%以上

### 6.2 成本结构

| 成本项 | 占比（预估） |
|--------|------------|
| AI API调用成本 | 30-40%（最大头） |
| 微信云开发资源 | 5-10% |
| 微信支付手续费 | 0.6% |
| 运营人力成本 | 15-20% |
| 其他（域名/工具等） | 5% |

**成本优化**：
- 利用微信混元免费额度（1万张文生图）降低初期AI成本
- 云开发选择按量付费，随用户量弹性扩缩

### 6.3 未来收入扩展（P2+）

| 收入来源 | 说明 |
|----------|------|
| 会员订阅 | 月度/年度VIP，每日额外积分 + 专属模型 + 去水印特权 |
| 广告收入 | 画廊信息流广告、创作等待页广告 |
| 品牌合作 | 定制AI模板/联名活动（如IP合作） |
| 企业版API | 批量AI生图服务，面向中小企业 |

## 7. 用户增长策略

### 7.1 冷启动阶段（0-1000用户）

- **种子用户**：朋友圈首发 + AI爱好者社群投放
- **新人大礼包**：注册送50积分，足够体验所有模型
- **画廊种子内容**：运营人员预置50-100张高质量AI作品，覆盖各分类
- **KOC合作**：邀请小红书/微博AI博主体验并发布评测

### 7.2 增长阶段（1000-10000用户）

- **邀请裂变**：邀请码双方各得20积分，生成邀请海报一键分享
- **社交分享**：作品自带小程序码水印，分享到朋友圈/群自动生成传播
- **画廊社区效应**：优质作品获得曝光 → "生成同款"降低门槛 → 更多人参与
- **每日签到**：培养用户打开习惯，连续签到奖励递增

### 7.3 留存策略

- 连续签到奖励递增（7天/30天里程碑奖励）
- 画廊新作品推送（微信订阅消息）
- 新模型上线通知
- 每周主题创作挑战（如"赛博朋克城市"、"水墨山水画"，获奖奖励积分）

### 7.4 变现阶段

- 免费积分耗尽 → 自然转化为付费用户
- 限时充值优惠（首充双倍、节日特惠）
- 高价模型效果对比展示（刺激升级消费）
- 会员体系引导（高频用户推荐月度VIP更划算）

### 7.5 北极星指标

| 指标 | 目标值 |
|------|--------|
| DAU / MAU | DAU 目标 MAU的20%+ |
| 次日留存率 | ≥ 40% |
| 7日留存率 | ≥ 20% |
| 付费转化率 | ≥ 5% |
| 日均生成图片数 | 随用户增长线性增长 |
| 画廊作品发布率 | 活跃用户中30%+发布过作品 |
| 邀请裂变系数（K-factor） | ≥ 0.3 |

## 8. UI/UX 设计规范

### 8.1 设计语言

- **风格**：现代简约 + 微渐变，突出AI科技感与创意氛围
- **圆角**：卡片统一 16rpx，按钮统一 8rpx
- **阴影**：轻柔阴影 `0 2rpx 8rpx rgba(0,0,0,0.06)`，层次分明

### 8.2 配色方案

#### 品牌色
| 用途 | 颜色 | 色值 |
|------|------|------|
| 品牌主色 | 极光紫 | `#6C5CE7` |
| 品牌辅色 | 赛博青 | `#00D2D3` |
| 渐变组合 | 紫→青 | `linear-gradient(135deg, #6C5CE7, #00D2D3)` |

#### 中性色
| 用途 | 色值 |
|------|------|
| 背景色 | `#F8F9FA` |
| 卡片背景 | `#FFFFFF` |
| 主文字 | `#2D3436` |
| 次文字 | `#636E72` |
| 辅助文字 | `#B2BEC3` |
| 分割线 | `#EEEEEE` |

#### 功能色
| 用途 | 色值 |
|------|------|
| 成功/点赞 | `#00B894`（绿） |
| 警告/积分 | `#FDCB6E`（金） |
| 错误/删除 | `#FF6B6B`（红） |
| 信息/链接 | `#74B9FF`（蓝） |

### 8.3 字体规范

| 用途 | 字号 | 字重 |
|------|------|------|
| 页面标题 | 34rpx | 600 |
| 卡片标题 | 30rpx | 500 |
| 正文 | 28rpx | 400 |
| 辅助文本 | 24rpx | 400 |
| 积分数字 | DIN Alternate 或系统数字字体 | 600 |

### 8.4 图标规范

- **TabBar图标**：81px × 81px（微信标准），线条风格，2px描边
- **功能图标**：统一使用 iconfont 或自绘 SVG
- 每个Tab有 `active`（极光紫填充）和 `normal`（灰色描边）两套状态

### 8.5 TabBar 设计

```
┌──────────────────────────────────────────────────────┐
│                                                       │
├────────┬────────┬────────┬────────┬───────────────────┤
│        │        │ ╭────╮ │        │                   │
│  🖼️   │  💡   │ │ ✨ │ │  📁   │       👤         │
│  画廊  │  灵感  │ ╰────╯ │  作品  │       我的        │
│ #B2BEC3│ #B2BEC3│  创作  │ #B2BEC3│     #B2BEC3      │
│        │        │渐变突出 │        │                   │
└────────┴────────┴────────┴────────┴───────────────────┘
```

- 背景：白色 + 顶部 1rpx 分割线 `#EEEEEE`
- 选中态：品牌主色 `#6C5CE7`
- 未选中态：`#B2BEC3`
- **创作Tab**：
  - 图标放大至 100px（其他81px）
  - 渐变色圆形背景（`#6C5CE7` → `#00D2D3`）
  - 白色图标（魔法棒/画板）
  - 视觉上突出于TabBar平面

### 8.6 交互规范

| 交互 | 规范 |
|------|------|
| 页面切换 | 默认右滑返回 |
| 图片加载 | 渐进式加载（模糊占位→清晰） |
| 生成等待 | 趣味动画 + 预估剩余时间 + 进度提示文案 |
| 下拉刷新 | 顶部 loading 动画 |
| 列表加载 | 骨架屏占位 |
| 弹窗 | 底部弹出式为主（ActionSheet 风格） |
| Toast | 轻量提示，2秒自动消失 |
| 空状态 | 每个列表页设计专属空状态插画 |

### 8.7 组件规范

| 组件 | 样式 |
|------|------|
| 主按钮 | 渐变色填充（紫→青），白色文字，圆角 8rpx |
| 次按钮 | 白色背景 + 极光紫描边，极光紫文字 |
| 文字按钮 | 无背景，极光紫文字 |
| 禁用按钮 | 灰色背景，浅灰文字 |
| 图片卡片 | 圆角 16rpx，轻阴影，底部信息栏 |
| 输入框 | 圆角 8rpx，浅灰背景，focus态紫色描边 |
