### 1.2 技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | WXML + WXSS + JavaScript | 微信原生开发，无框架依赖 |
| 后端 | 云函数 Node.js | 云开发内置运行环境 |
| 数据库 | 云数据库（文档型） | 类 MongoDB，支持索引和聚合 |
| 存储 | 云存储 | 图片/文件存储，支持CDN |
| AI服务 | REST API | 各厂商HTTPS接口 |
| 支付 | 微信支付 | 云开发内置支付能力 |

### 1.3 云函数架构策略

- **按业务域拆分**：避免单一巨型函数，每个云函数职责单一
- **路由模式**：每个云函数内部使用 `event.type` 分发到子处理函数
- **共享模块**：公共逻辑抽取为共享代码（AI适配器、工具函数、常量配置）

## 2. 数据库设计

## 3. 云函数设计

### 3.1 云函数拆分方案（7个）

#### 3.1.1 userAuth（用户认证与信息管理）

| action | 说明 |
|--------|------|
| `register` | 新用户注册：获取openid → 创建用户记录 → 赠送初始积分 → 生成邀请码 |
| `getUserInfo` | 获取用户信息 |
| `updateProfile` | 更新昵称/头像/签名 |
| `getStats` | 获取用户统计数据（总创作/发布/获赞） |

#### 3.1.2 creation（AI创作核心）

| action | 说明 |
|--------|------|
| `createImage` | AI生图主流程：参数校验 → 扣减积分 → 调用AI API → 存储结果 → 返回图片 |
| `getCreation` | 获取单条创作记录 |
| `getCreationList` | 获取创作历史（分页） |
| `deleteCreation` | 删除创作记录 |
| `getModels` | 获取可用模型列表（读取 models_config） |

#### 3.1.3 gallery（画廊社区）

| action | 说明 |
|--------|------|
| `publishPost` | 发布作品到画廊 |
| `getPostList` | 获取画廊列表（支持分类/排序/分页） |
| `getPostDetail` | 获取帖子详情 |
| `deletePost` | 删除帖子 |
| `toggleLike` | 点赞/取消点赞（原子操作更新 like_count） |
| `toggleCollect` | 收藏/取消收藏（原子操作更新 collect_count） |
| `remixCreation` | 生成同款：获取原Prompt和参数 |

#### 3.1.4 points（积分系统）

| action | 说明 |
|--------|------|
| `getBalance` | 获取积分余额 |
| `getLedger` | 获取积分明细（分页） |
| `signIn` | 每日签到：校验日期 → 计算连续天数 → 发放积分 |
| `getSignInCalendar` | 获取当月签到日历 |
| `createRechargeOrder` | 创建充值订单（调用微信支付） |
| `handlePaymentCallback` | 微信支付回调处理（加积分 + 写流水） |
| `redeemInviteCode` | 使用邀请码：校验有效性 → 双方发积分 |

#### 3.1.5 tools（AI图片处理工具）

| action | 说明 |
|--------|------|
| `processImage` | 统一图片处理入口：根据 tool_id 分发到不同处理逻辑 |
| `getToolList` | 获取工具列表（读取 ai_tools） |

支持的 tool_id：`cutout`（抠图）、`watermark_remove`（去水印）、`enhance`（变清晰）、`restore`（老照片修复）、`style_transfer`（风格迁移）、`outpaint`（扩图）

#### 3.1.6 admin（管理后台）

| action | 说明 |
|--------|------|
| `manageTemplates` | 模板 CRUD |
| `manageModels` | 模型配置管理 |
| `manageGallery` | 画廊内容审核/精选设置 |
| `getSystemStats` | 系统统计数据（用户数/创作数/收入） |
| `updateConfig` | 更新系统配置 |

#### 3.1.7 scheduled（定时任务）

| action | 说明 |
|--------|------|
| `dailyReset` | 每日重置（混元免费额度计数等） |

### 3.3 调用规范

**请求格式**（所有云函数统一）：
```javascript
wx.cloud.callFunction({
  name: 'creation',
  data: {
    type: 'createImage',   // action名称
    data: {                // 业务参数
      prompt: '一只可爱的猫咪',
      model_id: 'nano-banana-2',
      aspect_ratio: '1:1',
      count: 1
    }
  }
})
```

**响应格式**（统一）：
```javascript
{
  code: 0,          // 0=成功，非0=错误码
  message: 'success',
  data: { ... }     // 业务数据
}
```

### 3.4 错误码体系

| 范围 | 类别 |
|------|------|
| `0` | 成功 |
| `1001-1999` | 参数错误（缺少必填、格式错误、值越界） |
| `2001-2999` | 权限错误（未登录、无权限、非管理员） |
| `3001-3999` | 业务错误（积分不足、频率限制、邀请码无效、已签到） |
| `4001-4999` | AI服务错误（API调用失败、超时、模型不可用） |
| `5001-5999` | 系统错误（数据库异常、存储异常） |

## 4. AI模型API对接方案

### 4.1 统一适配层设计

所有AI模型通过适配器模式统一封装，每个Provider实现相同接口：

```javascript
// provider-factory.js
class AIProvider {
  // 文生图
  async generateImage(prompt, negativePrompt, params) {
    // 返回 { images: [{ base64, width, height }] }
  }
  // 图生图（可选）
  async editImage(imageBase64, prompt, params) {
    // 返回 { images: [{ base64, width, height }] }
  }
  // 获取模型能力
  getModelCapabilities() {
    // 返回 { text2img: true, img2img: false, edit: false }
  }
}

// 工厂函数
function getProvider(modelId) {
  const providerMap = {
    'nano-banana-2':   GoogleProvider,
```

### 4.6 统一错误处理

```javascript
async function callAI(modelId, prompt, params) {
  const provider = getProvider(modelId)
  const maxRetries = 2
  let lastError

  for (let i = 0; i <= maxRetries; i++) {
    try {
      const result = await provider.generateImage(prompt, params.negativePrompt, params)
      return result
    } catch (error) {
      lastError = error
      if (i < maxRetries) {
        await sleep(1000 * (i + 1)) // 指数退避
      }
    }
  }

  // 降级：返回友好错误信息
  throw new AIError(4001, `AI服务暂时不可用，请稍后重试或切换其他模型`)
}
```

## 6. 安全与风控

### 6.1 接口安全

| 措施 | 说明 |
|------|------|
| openid校验 | 所有云函数入口通过 `cloud.getWXContext()` 获取openid，不信任前端传入 |
| 参数校验 | 统一 `validator.js` 校验层，防注入攻击 |
| 频率限制 | 生图接口：单用户每分钟最多5次；签到/邀请：自然频率限制 |
| 积分原子操作 | 使用数据库事务或 `inc()` 原子操作，防并发超扣 |

### 6.2 AI API密钥管理

- 所有密钥存储在**云函数环境变量**中
- **不暴露到前端代码**
- 定期轮换

环境变量清单：
```
GOOGLE_AI_API_KEY
OPENAI_API_KEY
BYTEDANCE_ACCESS_KEY
BYTEDANCE_SECRET_KEY
```

### 6.3 内容安全

| 环节 | 措施 |
|------|------|
| Prompt审核 | 调用微信内容安全API `msgSecCheck` 校验输入 |
| 图片审核 | 调用微信图片安全API `imgSecCheck` 校验生成结果 |
| 画廊发布 | 强制内容审核，违规内容自动下架 |
| 违规记录 | 多次违规用户自动封禁 |

### 6.4 积分安全

- 积分扣减在**云函数端完成**，前端仅展示
- 使用数据库事务保证扣减原子性
- 积分流水完整记录，可审计追溯
- 异常检测：单用户短时间内大量生成 → 告警 + 限流

### 6.5 防刷策略

| 场景 | 防刷措施 |
|------|----------|
| 邀请码 | 同设备/同openid仅可使用一次 |
| 签到 | 服务端校验日期，不可伪造 |
| 新用户赠送 | 同设备指纹限制 |
| 微信支付 | 回调验签处理 |
| 画廊点赞/收藏 | 唯一索引防重复 |

### 6.6 图片安全

- 画廊图片**加水印后展示**
- 图片URL使用云存储fileID，不暴露原始路径
- 临时URL设置过期时间
- 用户上传的图片需经过安全检测

```
4. 云存储 → 按目录规划创建文件夹
```
